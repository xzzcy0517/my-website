## 3. 欢迎页设计

### 3.1 视觉系统

#### 配色方案

```
暗黑底色
  bg-primary:    #08080c  (最深黑)
  bg-secondary:  #111118  (卡片底)
  bg-tertiary:   #1a1a24  (悬浮态)

生活养生 — 暖金系
  accent-warm:        #d4a574  (主强调)
  accent-warm-glow:   rgba(212, 165, 116, 0.3)

科技 AI — 冷色系
  accent-cool:        #5eead4  (主强调)
  accent-cool-glow:   rgba(94, 234, 212, 0.3)

中性色
  text-primary:       #e4e4e7  (正文)
  text-secondary:     #a1a1aa  (辅助文字)
  border:             rgba(255,255,255,0.06)
  border-hover:       rgba(255,255,255,0.12)
```

#### 设计原则

- **少即是多**：每屏信息量克制，大量留白
- **光影感**：微妙的渐变光晕 + 毛玻璃卡片 + 细边框
- **对比克制**：不刺眼，所有颜色在暗黑底上收一档饱和度
- **温暖与冷静并存**：暖金和冷色在页面不同区域交替出现，映射两个内容方向

#### 交互细节表

| 元素 | 触发条件 | 动画效果 | 时长 |
|------|----------|----------|------|
| 标题 | 页面加载 | fade-in + translateY(-20px→0) | 0.8s |
| 副标题 | 标题完成后 | fade-in + 金色到蓝色渐变流动 | 0.6s |
| 自我介绍 | 滚动到该区域 (IntersectionObserver) | 打字机逐字出现 + 光标闪烁 | ~5-8s |
| 平台图标 | 进入视口，依次弹入 | stagger each 0.1s + spring 弹性 | 0.5s/个 |
| 平台图标 | hover | scale(1.08) + 外发光 8px + 边框变亮 | 0.2s |
| 平台图标 | click | 新标签页打开平台主页 | — |
| 微信按钮 | click | 从图标位置展开模态框 (layoutId 动画) | 0.3s |
| 模态框遮罩 | 出现 | backdrop-filter: blur(8px) + 暗色渐变 | 0.3s |
| 模态框内容 | 遮罩完成后 | scale(0.9→1) + fade-in | 0.25s |
| 模态框关闭 | 点遮罩 / 按 ESC / 点关闭按钮 | 反向动画 | 0.2s |

### 3.5 响应式断点设计

| 断点 | 宽度 | 设计行为 |
|------|------|----------|
| **Desktop** | ≥1280px | 标准布局，平台图标 5 个一行，最大宽度 1200px 居中 |
| **Laptop** | 1024-1279px | 容器宽度 90vw，字号缩小 5%，图标间距收紧 |
| **Tablet** | 768-1023px | 自我介绍字号缩小，平台图标 3+2 两行排列 |
| **Mobile** | <768px | 单列垂直布局，标题字号 2xl，自我介绍字号 base，平台图标每行 2 个，模态框全屏，粒子密度降低 |

#### 移动端专项优化

- 粒子 Canvas 在 `<768px` 时粒子数量减半
- 平台图标最小触摸区域 44×44px（WCAG 无障碍标准）
- 模态框在移动端改为底部抽屉式（Bottom Sheet）
- 滚动视差效果在移动端禁用
- 字体大小使用 `clamp()` 做流式缩放

## 6. 部署方案

一期只有静态文件，部署极简：

```bash
# 1. 构建
cd xiaochen-site
pnpm build --filter=@xiaochen/web
# → 产物在 apps/web/out/

# 2. 上传到服务器
scp -r apps/web/out/* user@server:/opt/site/

# 3. Nginx 配置
```

Nginx 极简配置：

```nginx
server {
    listen 80;
    server_name xzzcy.com;

    location / {
        root /opt/site;
        index index.html;
        try_files $uri $uri.html $uri/ =404;
    }

    # 静态资源长期缓存
    location /_next/static/ {
        root /opt/site;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 图片缓存
    location /images/ {
        root /opt/site;
        expires 7d;
        add_header Cache-Control "public";
    }
}
```

> 无需 MySQL、Redis、Node 运行时、PM2。Nginx 直接托管静态文件即可。
