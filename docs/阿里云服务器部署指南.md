# 阿里云服务器部署指南 — 从零到上线

本指南将帮你把「最喜欢真真的小陈」个人网站部署到阿里云服务器上，让其他人可以通过 IP 地址访问。

**部署方式**：服务器配置 SSH 密钥 → 克隆 GitHub 仓库 → 服务器上构建打包 → Nginx 代理访问。

---

## 第 0 步：选择服务器操作系统

在阿里云镜像列表中找到 **Ubuntu 22.04**（注意：选**不带 UEFI 字样**的普通版本），点击确认即可。

镜像选好后，顺手设置以下三项：

- **系统盘**：默认 40GB 够用了，不够以后可以扩容
- **root 密码**：设一个你自己记得住的密码（本文后面会用到）
- **实例名称**：随便填，比如 `my-website`

## 你的项目概况

| 项目 | 详情 |
|------|------|
| 类型 | Next.js 15 静态网站（纯 HTML/CSS/JS） |
| 框架 | React 19 + Tailwind CSS v4 + Framer Motion |
| 构建工具 | Turborepo + pnpm |
| 构建产物 | `apps/web/out/` 目录下的静态文件 |
| 服务器 | 阿里云 ECS，2核2G |
| 服务器 IP | `120.76.43.51` |
| 操作系统 | Ubuntu 22.04（[第 0 步](#第-0-步选择服务器操作系统)有详细选型说明） |
| 域名 | 暂无 |

> **好消息**：你的项目是纯静态网站，不需要数据库，不需要 PM2 进程守护。构建出静态文件后只需要一个 Nginx 就能跑起来。

## 整体流程（7 步）

```
控制台登录服务器 → 安装基础工具 → 配置SSH密钥(服务器→GitHub) → 安装Node.js+pnpm
→ 克隆仓库+构建 → 配置防火墙 → 安装配置Nginx → 访问验证 ✅
```

> 📌 本文所有操作都在**阿里云控制台网页终端**中完成，不需要在 Mac 上安装任何工具。

## 第 1 步：通过阿里云控制台登录服务器

### 1.1 进入网页终端

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 左侧菜单 → **实例** → 找到你的服务器实例
3. 点击实例右侧的 **「远程连接」** 按钮
4. 在弹出的窗口中，选择 **「通过 VNC 登录」** 或 **「Workbench 远程连接」**（推荐 Workbench）
5. 点击 **「立即登录」**

> 如果提示需要密码，输入 root 密码即可。如果忘记密码，可以在实例详情页点击「重置实例密码」。

### 1.2 登录成功

登录成功后，你会看到类似这样的命令行提示符：

```
root@iZxxxxx:~#
```

这意味着你已经以 `root` 身份进入了服务器，接下来所有的命令都在这个网页终端中执行。

## 第 2 步：服务器基础初始化

### 2.1 更新系统软件包

```bash
apt update && apt upgrade -y
```

这一步会更新所有系统软件包到最新版本，大约需要 2-5 分钟。

更新过程中可能会弹出一个紫色界面的提示：

```
Daemons using outdated libraries
Which services should be restarted?
    [*] packagekit.service
         <Ok>              <Cancel>
```

这是系统在问你"有些服务需要重启，要现在重启吗？"**按 Tab 键选中 `<Ok>`，然后按回车**即可。这是正常流程，不会影响任何东西。

### 2.2 安装必要工具

```bash
apt install -y curl wget git nano unzip
```

验证 Git 安装成功：

```bash
git --version
# 应输出类似 git version 2.43.0
```

## 第 3 步：配置服务器 SSH 密钥（连接 GitHub）

这一步是关键——让服务器可以通过 SSH 免密拉取你的 GitHub 仓库。

### 3.1 在服务器上生成 SSH 密钥

```bash
# 使用 ed25519 算法（推荐，更安全更短）
ssh-keygen -t ed25519 -C "xzzcy0517-aliyun-server"

# 所有提示都直接按回车，使用默认路径和空密码短语：
#   Enter file in which to save the key → 回车
#   Enter passphrase → 回车
#   Enter same passphrase again → 回车
```

生成成功后，查看公钥内容：

```bash
cat ~/.ssh/id_ed25519.pub
```

输出类似：
```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIxxxxxxxxxxxxxxxxxxxxxx xzzcy0517-aliyun-server
```

**复制这整行内容**（在网页终端中用鼠标选中后按 `Ctrl+C` 复制，或者直接鼠标右键复制）。

### 3.2 把服务器公钥添加到 GitHub

1. 在浏览器打开 [GitHub → Settings → SSH and GPG keys](https://github.com/settings/keys)
2. 点击绿色按钮 **「New SSH key」**
3. **Title**：填写 `阿里云服务器`（方便以后识别）
4. **Key type**：保持 `Authentication Key`
5. **Key**：粘贴刚才复制的那一整行公钥
6. 点击 **「Add SSH key」**

### 3.3 测试 SSH 连接 GitHub

在服务器上执行：

```bash
ssh -T git@github.com
```

首次连接会提示确认 GitHub 的主机指纹：
```
The authenticity of host 'github.com (140.82.121.4)' can't be established.
...
Are you sure you want to continue connecting (yes/no/[fingerprint])?
```

输入 `yes` 回车。

如果成功，会显示：
```
Hi 你的用户名! You've successfully authenticated, but GitHub does not provide shell access.
```

看到这行字就说明服务器已经可以通过 SSH 免密访问 GitHub 了！✅

> ⚠️ 如果显示 `Permission denied (publickey)`，回到 3.1 步检查公钥是否完整复制到了 GitHub。

## 第 5 步：克隆仓库并构建

### 5.1 克隆仓库到服务器

```bash
# 进入 /opt 目录（服务器上放项目的常用位置）
cd /opt

# 使用 SSH 地址克隆（重要！用 SSH 不用 HTTPS）
git clone git@github.com:你的GitHub用户名/my-website.git

# 进入项目目录
cd my-website
```

> 📌 把 `你的GitHub用户名` 替换为你的实际 GitHub 用户名。SSH 地址格式是 `git@github.com:用户名/仓库名.git`，可以在 GitHub 仓库页面 → 绿色「<> Code」按钮 → SSH 标签中复制。

### 5.2 安装依赖

```bash
pnpm install
```

这一步会下载所有项目依赖，服务器在国内，网络通常没问题。大约需要 1-3 分钟。

### 5.3 构建项目

```bash
pnpm build
```

构建成功后，你会看到类似输出：
```
✓ Compiled successfully
✓ Export successful
```

构建产物在 `apps/web/out/` 目录下。确认一下：

```bash
ls apps/web/out/
# 应该看到：404.html  index.html  robots.txt  images/  _next/
```

## 第 6 步：配置防火墙

先把防火墙打通，这样后面装 Nginx 的时候可以直接在浏览器验证。

阿里云有**两层防火墙**，两层都要配置，这是新手最容易卡住的地方！

### 6.1 阿里云安全组（第一层 —— 云端防火墙）

阿里云安全组默认只开放了 22（SSH）和 3389（远程桌面）端口，需要手动放行 80 端口。

**操作步骤**：

1. 登录 [阿里云控制台](https://ecs.console.aliyun.com/)
2. 左侧菜单 → **网络与安全** → **安全组**
3. 找到你实例绑定的安全组，点击 **配置规则**
4. 点击 **入方向** → **手动添加**
5. 添加以下规则（注意：协议类型选 **自定义 TCP**，不要选"所有 TCP"）：

| 优先级 | 协议类型 | 端口范围 | 授权对象 | 描述 |
|--------|----------|----------|----------|------|
| 1 | 自定义 TCP | 80 | 0.0.0.0/0 | HTTP |
| 1 | 自定义 TCP | 443 | 0.0.0.0/0 | HTTPS（备用） |

> 选**自定义 TCP** 然后单独填端口号，而不是"所有 TCP"——"所有 TCP" 会把服务器所有端口都暴露到公网，非常不安全。
>
> `0.0.0.0/0` 表示允许所有 IP 访问。你的网站是给公众看的，用这个没问题。

6. 点击 **保存**

### 6.2 系统防火墙（第二层 —— 服务器自带）

```bash
# 检查防火墙状态
ufw status

# 如果显示 Status: inactive，说明防火墙没开，跳过本小节即可
# 如果显示 Status: active，则需要放行以下端口：
ufw allow 80/tcp
ufw allow 443/tcp
ufw allow 22/tcp     # 确保 SSH 端口不被封
ufw reload
```

## 第 7 步：安装并配置 Nginx

### 7.1 安装 Nginx

```bash
apt install -y nginx
```

验证安装：

```bash
nginx -v
# 应输出 nginx version: nginx/1.24.0
```

### 7.2 启动 Nginx 并在浏览器验证

```bash
# 启动 Nginx 并设置开机自启
systemctl start nginx
systemctl enable nginx

# 确认 Nginx 正在运行
systemctl status nginx
# 看到 active (running) 就对了
```

现在打开浏览器，访问：

```
http://120.76.43.51
```

如果看到 **Welcome to nginx!** 的页面，说明 Nginx 已经跑起来了 ✅

> 防火墙已经在第 6 步配好了，所以浏览器直接就能访问。

### 7.3 把构建产物拷贝到 Nginx 目录

```bash
# 清空默认目录
rm -rf /var/www/html/*

# 把构建产物拷贝到 Nginx 站点目录
cp -r /opt/my-website/apps/web/out/* /var/www/html/
```

### 7.4 创建站点配置文件

```bash
nano /etc/nginx/sites-available/xiaochen
```

将以下内容完整粘贴进去（在网页终端中粘贴通常是 `Ctrl+Shift+V` 或右键粘贴）：

```nginx
server {
    listen 80;
    server_name 120.76.43.51;

    root /var/www/html;
    index index.html;

    # Gzip 压缩——让网站加载更快
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml application/xml+rss text/javascript;
    gzip_min_length 256;
    gzip_vary on;

    # 静态资源缓存——带 hash 的 JS/CSS 缓存一年
    location /_next/static/ {
        alias /var/www/html/_next/static/;
        expires 1y;
        add_header Cache-Control "public, immutable";
    }

    # 图片缓存 30 天
    location /images/ {
        alias /var/www/html/images/;
        expires 30d;
        add_header Cache-Control "public";
    }

    # 主路由——尝试匹配 .html 文件，最后回退到 index.html
    location / {
        try_files $uri $uri.html $uri/ /index.html;
    }

    # 自定义 404 页面
    error_page 404 /404.html;
    location = /404.html {
        internal;
    }
}
```

粘贴完成后，按 `Ctrl+X`（退出）→ 按 `Y`（确认保存）→ 按**回车**（确认文件名），搞定。

### 7.5 启用站点并重载 Nginx

```bash
# 创建软链接，启用站点
ln -sf /etc/nginx/sites-available/xiaochen /etc/nginx/sites-enabled/

# 删除默认配置（避免冲突）
rm -f /etc/nginx/sites-enabled/default

# 测试 Nginx 配置是否正确
nginx -t

# 应该显示：
#   nginx: the configuration file ... syntax is ok
#   nginx: configuration file ... test is successful

# 重载 Nginx 让新配置生效
systemctl reload nginx
```

## 以后怎么更新网站？

你本地的修改推送到 GitHub 之后，在服务器上只需要三步：

### 一键更新脚本（推荐）

在服务器上创建一个更新脚本：

```bash
nano /opt/update-site.sh
```

粘贴以下内容：

```bash
#!/bin/bash
set -e
echo "📥 拉取最新代码..."
cd /opt/my-website
git pull

echo "🔨 重新构建..."
pnpm build

echo "📦 更新站点文件..."
rm -rf /var/www/html/*
cp -r apps/web/out/* /var/www/html/

echo "✅ 更新完成！访问 http://120.76.43.51"
```

粘贴完成后，按 `Ctrl+X` → `Y` → 回车 保存。

```bash
# 赋予执行权限
chmod +x /opt/update-site.sh
```

以后每次更新，通过阿里云控制台远程连接登录服务器，然后执行：

```bash
/opt/update-site.sh
```

## 附录：SSH 密钥泄露后如何重建

如果你的服务器 SSH 密钥已经泄露（比如不小心把私钥发给了别人、密钥出现在了公开仓库中），需要立即更换。以下是完整操作步骤。

### 第一步：在 GitHub 上删除旧公钥

1. 登录 GitHub → 右上角头像 → **Settings**
2. 左侧菜单 → **SSH and GPG keys**
3. 找到之前添加的密钥（Title 为「阿里云服务器」或类似名称）
4. 点击右侧的 **Delete** → 确认删除

> ⚠️ 这一步最关键——先废掉旧公钥，防止别人用泄露的私钥冒充你的服务器。

### 第二步：在服务器上删除旧密钥

在服务器上执行：

```bash
# 删除旧的 SSH 密钥对（私钥 + 公钥）
rm -f ~/.ssh/id_ed25519 ~/.ssh/id_ed25519.pub

# 如果用的是 RSA 密钥：
# rm -f ~/.ssh/id_rsa ~/.ssh/id_rsa.pub
```

### 第三步：在服务器上生成新密钥

```bash
# 生成新的 ed25519 密钥（注意改一下注释里的标识，方便区分新旧）
ssh-keygen -t ed25519 -C "xzzcy0517-aliyun-server-new"

# 所有提示直接回车：
#   Enter file in which to save the key → 回车
#   Enter passphrase → 回车（或设置密码）
#   Enter same passphrase again → 回车
```

### 第四步：把新公钥添加到 GitHub

```bash
# 查看并复制新公钥
cat ~/.ssh/id_ed25519.pub
```

1. 回到 GitHub → **Settings** → **SSH and GPG keys**
2. 点击 **New SSH key**
3. Title 填写 `阿里云服务器（新）`
4. 粘贴公钥 → 点击 **Add SSH key**

### 第五步：测试新密钥是否生效

```bash
ssh -T git@github.com
```

看到 `Hi 你的用户名! You've successfully authenticated...` 即表示新密钥生效 ✅

### 第六步：验证服务器仍能正常拉取代码

```bash
cd /opt/my-website
git pull
```

能正常拉取说明一切正常，旧密钥已彻底废弃。

### 安全建议

- 以后每次生成密钥时，注释（`-C` 后面的内容）加一个日期或版本号，方便识别
- 定期检查 GitHub 的 [SSH keys 页面](https://github.com/settings/keys)，清理不再使用的旧密钥
- 私钥文件（`id_ed25519`，不带 `.pub` 后缀）**永远不要**发给任何人、不要上传到任何地方
